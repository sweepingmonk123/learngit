== 产品介绍
袋鼠云实时大数据智能操作系统(Real Big Data Operating System：RD-OS)，是一款高效的大数据实时/离线任务开发、任务调度、数据管理工具，支持对大数据实时处理过程进行可视化管理与控制。帮助客户提升开发效率，快速创建实时/离线计算任务，缩短开发周期；任务管理与运维一体化，减轻繁冗的运维工作。

* **主要特性**

//. 数据源：支持结构化数据、半结构化数据集成至RD-OS平台，可对接关系型数据库（MySQL、Oracle等）、HDFS或HBase等存储单元。
//. 数据集成：支持对各离线数据源的全量读取和写入，对关系型数据库支持增量读取（通过过滤规则控制），支持从kafka读取流式数据，并进行实时计算。
//. 资源管理：支持YARN作为统一资源调度管理系统。
//. 任务开发：统一的实时/离线任务工作台
//.. 实时任务开发：支持可视化创建基于MR或SQL的实时任务，并支持配置任务执行参数；
//.. 离线数据查询：支持以WEB的形式编写SQL（Hive）实现离线数据查询并显示；
//.. 离线任务开发：支持可视化创建离线任务（Spark）；支持配置任务调度周期；支持可视化配置离线任务间的调度依赖关系，设定各个任务执行的先后顺序；
//.. 资源管理：支持实时/离线任务的资源上传、基本信息编辑、删除等；
//. 任务运维：统一的运维管理入口
//.. 实时任务运维监控：统一查看所有任务运行状态，查看错误日志，进行任务状态转换操作（启动/停止）；
//.. 离线任务运维监控：统一查看任务运行装填，查看执行日志，支持手动重跑等功能；
//.. 监控报警：自定义监控规则，及时获悉异常信息
//... 报警信息提醒：各个任务触发报警规则后，自动向责任人发送邮件/短信通知；
//... 自定义报警规则：支持自定义报警规则配置，可设定任务状态、接收方式、接收人等；
//. 数据管理：查看平台上的所有数据表，包括占用存储情况、字段信息、分区信息、查看历史记录等；
//. 项目管理：3类系统角色（项目所有者、项目管理员、普通项目成员）对各个任务具有不同的操作权限，提升数据处理过程的可靠性；
//. 多租户管理：支持多租户管理，不同租户间实现数据隔离，保证数据安全；

* 全面托管的调度

提供强大的调度能力，支持按照时间、依赖关系的任务触发机制，支持各类任务按照DAG关系准确、准时运行。支持分钟、小时、天、周和月多种调度周期配置。
完全托管的服务，无需关心调度服务器资源问题。租户之间提供隔离，保证不同租户之间的任务不会相互影响。

* 支持多种任务类型

支持数据同步、SQL（兼容Hive语法）、MR、Python等多种任务类型，通过任务之间的相互依赖完成复杂的数据分析处理。数据同步模块支持多种数据源，提供稳定高效的数据传输。

* 可视化开发

提供可视化的代码开发页面，无需搭配任何开发工具，只要有浏览器有网络，便可随时随地进行开发工作。

* 监控告警

运维中心提供可视化的任务监控管理工具，支持以DAG图的形式展示任务运行时的全局情况。可方便地配置短信报警，任务发生错误可及时通知相关同学，保证业务正常运行。

* 约束与限制

仅支持Chrome浏览器65以上版本。

== 用户角色

[cols="1,4", options="header"]
|===
|角色
|权限
|项目所有者
|指项目空间的创建者，拥有项目空间内的最高权限，可对该项目空间的基本属性、数据源、当前项目空间的全部资源和项目成员等进行管理，并为项目成员赋予项目管理员、开发、运维、访客角色。
|项目管理员
|指项目空间的管理者，可对该项目空间的基本属性、数据源、当前项目空间的全部资源和项目成员等进行管理，并为项目成员赋予项目管理员、开发、运维、访客角色。项目管理员不能对项目所有者操作。
|数据开发
|数据开发角色的用户能够创建任务、脚本、资源等，新建/删除表，但不能执行发布操作。
|运维
|运维角色的用户由项目管理员分配运维权限；拥有发布任务、运维任务的操作权限，没有数据开发的操作权限。
|访客
|只具备查看权限，没有权限进行编辑任务、运维等操作
|===


NOTE: 目前RD-OS的用户角色是固定的，不支持编辑，每个角色详细的权限点可参考<<Appendix>>。
